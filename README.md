# LLM 뉴스레터 자동화 에이전트

네이버 뉴스 API, 기사 본문 크롤링, PostgreSQL, 로컬 LLM(Ollama), LangChain, SMTP를 활용해 특정 인물의 뉴스를 자동으로 수집하고 요약한 뒤, 유사 이슈를 묶어 HTML 뉴스레터로 발송하는 자동화 프로젝트입니다.

현재는 `로이킴` 키워드를 기준으로 동작하며, 향후에는 특정 인물 또는 관심 주제를 지속적으로 모니터링하는 AI Agent의 뉴스 브리핑 기능으로 확장하는 것을 목표로 합니다.

## 프로젝트 목적

RPA 개발 경험을 바탕으로 반복적인 정보 수집 업무를 LLM 기반 자동화 파이프라인으로 확장했습니다.

단순히 뉴스를 가져오는 데서 끝나지 않고, 기사 본문 수집, 로컬 LLM 요약, 유사 뉴스 그룹핑, 뉴스레터 생성, 메일 발송까지 하나의 end-to-end 흐름으로 구성했습니다.

## 주요 기능

- 네이버 뉴스 API 기반 키워드 뉴스 수집
- 날짜 범위 기준 일일 뉴스 필터링
- 기사 본문 크롤링
- PostgreSQL 기반 뉴스 데이터 저장
- 중복 기사 저장 방지
- 로컬 LLM 기반 기사 요약
- 특정 인물 관련 내용 중심 요약
- LLM 기반 유사 뉴스 그룹핑
- LLM 그룹핑 결과 검증 후 DB 업데이트
- HTML 뉴스레터 생성
- SMTP 기반 메일 발송
- 전체 실행 실패 시 에러 로그 저장

## 데이터베이스 설정

PostgreSQL에서 `news` 데이터베이스를 생성한 뒤 `schema.sql`을 실행합니다.

```sql
CREATE DATABASE news;
```

그 다음 `schema.sql`의 테이블 생성 쿼리를 실행합니다.

## 실행 방법

```bash
python main.py
```

현재 구현은 실행일 기준 전날 00:00:00부터 실행일 00:00:00 전까지의 뉴스를 처리합니다.

예를 들어 2026년 6월 29일에 실행하면 다음 범위를 처리합니다.

```text
2026-06-28 00:00:00 <= pub_date < 2026-06-29 00:00:00
```

## LLM 활용 방식

### 1. 기사 요약

기사 전체 내용과 특정 인물명이 등장한 주변 문맥을 함께 전달해, 기사 전체 핵심 이슈와 특정 인물 관련 언급을 분리해 요약합니다.

### 2. 유사 뉴스 그룹핑

요약된 뉴스 목록을 LLM에 전달해 같은 이슈끼리 뉴스 ID를 그룹핑합니다.

LLM 출력은 DB에 바로 반영하지 않고, 다음 조건을 검증한 뒤 업데이트합니다.

- 입력된 모든 뉴스 ID가 포함되었는지
- 중복된 뉴스 ID가 없는지
- 존재하지 않는 뉴스 ID가 포함되지 않았는지

## 현재 개선 예정 사항

- 날짜 필터링 및 LLM 그룹 검증 로직 테스트 추가
- 특정 인물명을 환경변수 또는 CLI 인자로 분리
